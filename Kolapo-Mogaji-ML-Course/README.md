# Kolapo Mogaji — Machine Learning Course Portfolio

Welcome to my Machine Learning portfolio. This repository contains the labs and assignments I produced during the course, along with written reflections on each one. The point of the portfolio is to capture not just the working code but the learning that happened around it — the choices I made, the things that didn't work the first time, and what I took away from each piece of work.

---

## About Me

I'm a junior at Houston City College pursuing a degree in Artificial Intelligence and Robotics, graduating May 2027. My interest in machine learning sits at the intersection of three things I care about: practical engineering, applied AI for real-world problems, and clear communication about how those systems work.

Outside of coursework I run a creative production company where I've built ETL pipelines, client dashboards, and AI-powered tools. That work is what pulled me toward formal ML training in the first place — I kept hitting points where I needed to understand the models I was integrating with, not just call their APIs. This course gave me that foundation.

I came in wanting concrete experience with the ML pipeline end-to-end: data preparation, feature engineering, model selection, training, evaluation, and the parts that usually get left out of tutorials — interpreting results honestly, knowing when a model is wrong, and writing about the work in a way someone else can follow.

## How This Repository Is Organized

```
Kolapo-Mogaji-ML-Course/
├── README.md              ← you are here
├── LICENSE
├── .gitignore
├── Labs/
│   ├── Lab1/
│   │   ├── README.md      ← what I did, what I learned, challenges
│   │   └── (code)
│   ├── Lab2/
│   └── ...
└── Assignments/
    ├── Assignment1/
    │   ├── README.md      ← problem, approach, results, interpretation
    │   └── (code)
    ├── Assignment2/
    └── ...
```

Each lab and assignment lives in its own folder with a dedicated README explaining the work. The READMEs are written to be readable on their own — you don't have to dig through code to understand what each one was about.
